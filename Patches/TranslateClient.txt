###########################################################################
# This is the text file used for translations by the Translate Client patch.
# The patcher only recognizes three starting characters - rest are ignored
# So you can comment/draw whatever with any other character you want.
# Recognized starting characters are
#
# M: <= used for providing a message string (details below)
# F: <= String to find
# R: <= String to replace with
#
# Both F: & R: can recognize HEX as well as regular ASCII strings.
# To specify ASCII strings enclose them in single quotes.
# To specify HEX values specify them as byte by byte with space in between
#
# Want an example? just check any of the patterns below. you will see almost
# all of my find strings are HEX but replace strings are ASCII :D
#
# Patcher does not stop when it fails to find any of the strings. Instead 
# it will be reported in a text file called FailedTranslations.txt 
# with the message specified using M:#
#
###########################################################################
